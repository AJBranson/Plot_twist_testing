1. Enhanced Error Handling
Status: Mostly done

game-state.js
saveGame() and loadGame() both use try/catch.
They notify the user with friendly messages on failure.
wallet.js
connectWallet() handles missing window.platformSDK.
It notifies the user when wallet support is unavailable or anonymous.
leaderboard.js
Supabase submit and fetch methods use retry loops and catch errors.
They show a user-facing notify on failure.
Implementation notes

Additional edge-case coverage could still be added for wallet commands and marketplace API failures.
Centralize error/display behavior so all async operations use a consistent pattern.
2. Development Cleanup
Status: Not fully done

There are still debug/log statements in:
game-state.js (console.warn('Save failed:', e), console.warn('Load failed:', e))
wallet.js (console.log('Dev mode: no Metanet SDK...'))
leaderboard.js (console.warn('Leaderboard submit failed:'), console.warn('Leaderboard fetch failed:'))
marketplace.js (console.warn('Stand toggle error:', e))
Implementation notes

A small logging wrapper or DEBUG flag would make it easy to keep developer output while removing console noise in production.
Search for console.log / console.warn and decide which are essential vs. removable.

4. Performance Optimizations
Status: Partially done

The app supports selective rendering:
renderPlots()
renderStorage()
renderWateringCan()
renderCompost()
renderAll() still exists and is used frequently.
Implementation notes

Replace some renderAll() calls with more targeted render calls where state changes are local.
Consider adding a lightweight render scheduler or debounce for burst updates.
No virtual DOM implementation exists.

Implementation notes

Add a hidden debug panel or state inspector.

Practical Next Steps

wallet.js for wallet edge-case logic
leaderboard.js for Supabase retry and notification behavior

rendering.js + index.html for accessibility and ARIA enhancements
